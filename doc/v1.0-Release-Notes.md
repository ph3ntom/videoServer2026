## 🎉 개요

StreamFlix v1.0은 라즈베리파이에서 동작하는 개인용 스트리밍 플랫폼입니다. Netflix와 YouTube의 핵심 기능을 결합하여 태그 기반 검색, HLS 스트리밍, 호버 미리보기 등 현대적인 스트리밍 서비스의 모든 필수 기능을 제공합니다.

---

## ✨ 주요 기능

### 🔍 검색 & 발견
- **키워드 검색**: 제목/설명 전체 텍스트 검색
- **태그 조합 검색**: 포함/제외 조건으로 정밀 검색
  - 예: `korea + SF - action` → 한국 SF 영화 중 액션 제외
- **정렬 옵션**: 최신순, 인기순, 평점순, 시청수순
- **카테고리 브라우징**: 장르별 탐색

### 🎬 비디오 스트리밍
- **HLS (HTTP Live Streaming)**: 네트워크 상황에 따른 자동 화질 조정
- **다중 화질 지원**: 480p, 720p, 1080p, 4K
- **화질 선택 UI**: 플레이어 내장 메뉴 (YouTube 스타일)
- **HTTP Range Requests**: 빠른 시킹 (seeking) 지원
- **HLS 변환 진행률**: 실시간 변환 상황 표시

### 🖼️ 썸네일 시스템
- **자동 생성**: FFmpeg 장면 전환 감지로 10-15개 생성
- **호버 미리보기**:
  - **이미지 순환**: 1초 간격으로 12개 썸네일 순환
  - **비디오 클립**: 3초짜리 MP4 클립 7개 자동 재생
- **관리자 선택**: 원하는 썸네일을 메인 이미지로 설정
- **외부 이미지 업로드**: 포스터 이미지 등록 가능

### 👤 사용자 & 권한
- **JWT 인증**: 안전한 토큰 기반 인증
- **역할 관리**: user (일반), admin (관리자), premium (추후)
- **Admin 전용 기능**: 비디오 업로드/수정/삭제, 태그 관리, 썸네일 선택

### ⭐ 사용자 상호작용
- **평점 시스템**: 별점 1-5점
- **시청 기록**: 시청한 비디오 추적
- **이어보기**: 마지막 시청 위치에서 재개
- **시청 완료 표시**: 비디오 카드에 진행률 바 표시

### ⚡ 성능 최적화
- **DB 인덱스**: 검색 및 정렬 쿼리 최적화
- **프론트엔드 최적화**:
  - React.lazy로 코드 스플리팅
  - useMemo/useCallback로 리렌더링 최소화
  - Intersection Observer로 lazy loading
- **캐시 제어**: 브라우저 캐싱 방지 (no-cache headers)

## 📦 설치 및 실행

### 시스템 요구사항
- **서버**: 라즈베리파이 4B/5 (8GB RAM 권장)
- **스토리지**: 4TB 외장 HDD
- **OS**: Raspbian (Debian 기반)
- **Docker**: 24.0+
- **FFmpeg**: 6.0+

### 빠른 시작

#### 1. 환경 변수 설정
```bash
cp .env.example .env
# .env 파일 편집
```

#### 2. Docker 실행
```bash
docker-compose up -d
```

#### 3. 데이터베이스 마이그레이션
```bash
docker exec -it streamflix-backend alembic upgrade head
```

#### 4. 슈퍼 관리자 생성
```bash
docker exec -it streamflix-backend python create_admin.py
```

#### 5. 접속
```
http://localhost:8000        # 백엔드 API
http://localhost:3000        # 프론트엔드
```

자세한 설치 가이드는 [README.md](../README.md)를 참고하세요.

## 🐛 알려진 이슈

### 제한사항
1. **동시 접속**: 현재 최대 3명 (라즈베리파이 성능 제약)
2. **HLS 변환**: 고해상도 비디오는 시간 소요 (4K: ~30분)
3. **메모리**: 대용량 비디오 업로드 시 메모리 부족 가능

### 해결 방법
1. **동시 접속**: Redis 캐싱 추가 (v1.1)
2. **HLS 변환**: 백그라운드 큐 시스템 (Celery) 도입 (v1.2)
3. **메모리**: 청크 업로드 개선 (v1.1)
