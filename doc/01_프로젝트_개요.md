# 01. 프로젝트 개요

## 프로젝트 명
**StreamFlix** - 라즈베리파이 기반 경량 스트리밍 플랫폼

## 프로젝트 목표
라즈베리파이에서 안정적으로 동작하는 개인/소규모 조직용 비디오 스트리밍 플랫폼을 구축합니다.

## 핵심 가치 제안
1. **효율성**: 라즈베리파이 4B/5 (8GB)에서도 원활하게 동작
2. **직관성**: 태그 조합 검색으로 원하는 비디오를 빠르게 찾기
3. **심플함**: 필요한 핵심 기능만 구현하여 복잡도 최소화
4. **확장성**: 개인용(3명) → 소규모 조직(100명)까지 대응

## 주요 기능

### MVP (2개월) - 절대 필수
- ✅ **Admin 전용 비디오 등록** (일반 사용자 업로드 불가)
- ✅ **제목 키워드 검색** (1순위 검색 방법)
- ✅ **태그 조합 검색** (한국+SF-액션 형태)
  - 체크박스 + 연산자 선택 UI
- ✅ **썸네일 자동 생성** (장면 전환 감지 방식)
- ✅ **썸네일 호버 미리보기** (1초 간격, 자연스러운 애니메이션)
- ✅ **카테고리 브라우징**
- ✅ **격자 레이아웃 UI** (YouTube 스타일)
- ✅ **다크 모드** (Netflix 스타일 색상)

### Phase 2 (4개월) - 중요
- 평점 시스템 (별점 1-5)
- 시청 기록 및 이어보기
  - 시청 완료 표시 (우선)
  - 이어보기 기능
  - 시청 기록 삭제

### Phase 3 (6개월) - 선택적
- 다중 화질 지원 (480p, 720p, 1080p, 4K)
- 자막 지원 (한국어 SRT/VTT)
- Premium 역할 (프리미엄 콘텐츠)
- HTTPS 적용

### 제거된 기능 (불필요)
- ❌ 댓글 시스템
- ❌ 소셜 공유 기능
- ❌ 플레이리스트
- ❌ 다운로드 기능
- ❌ 라이브 스트리밍
- ❌ 실시간 알림
- ❌ AI 추천 (단순 정렬로 대체)

## 인프라 환경

### 서버
- **하드웨어**: 라즈베리파이 4B 또는 5 (8GB RAM)
- **OS**: 라즈비안 + CasaOS
- **스토리지**: 외장 하드 4TB (NAS)
- **배포**: Docker 단일 컨테이너 (올인원)

### 네트워크
- 외부 IP + 포트포워딩
- HTTPS (추후 적용)

### 성능 목표
- 초기 동시 접속: 3명
- 1년 후 목표: 100명
- 비디오 로딩 시간: < 3초
- 메모리 사용: < 2GB (백엔드 + DB)

## 콘텐츠 규모
- **초기**: 100개 비디오 (~100GB)
- **1년 후**: 5,000개 비디오 (~5TB)
- **평균 파일 크기**: 1GB
- **지원 형식**: mp4, mkv, avi, mov

## 대상 사용자
- **일반 사용자 (user)**: 비디오 시청 및 검색만 가능
- **관리자 (admin)**: 비디오 업로드, 관리, 태그 생성
- **프리미엄 (premium)**: 추후 프리미엄 콘텐츠 접근 (Phase 3)

## 프로젝트 제약사항
- ⚠️ 라즈베리파이 성능 제약 (트랜스코딩 최소화)
- ⚠️ 메모리 8GB 한정 (최적화 필수)
- ⚠️ 단일 Docker 컨테이너 (복잡도 최소화)
- ⚠️ 외부 접근 (보안 주의)
